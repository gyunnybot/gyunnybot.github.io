## 경로 설정 시 주의할 점
Windows 환경에서 cmd(명령 프롬프트)를 사용해 스프링 부트 프로젝트를 빌드하거나 실행할 때는 프로젝트 경로나 폴더 이름을 가급적 영문/ASCII로 두는 것이 권장된다.

Spring 프로젝트 빌드 도구인 Gradle 또는 Maven은 JVM 위에서 동작하는데, JDK 17까지는 실행 중인 자바 프로젝트가 레거시인 경우 OS의 기본 문자셋(한국어 Windows에서는 보통 MS949)을 따랐다. 반면, IntelliJ와 같은 최신 IDE나 JDK 18 이상의 javac, jar 개발 툴은 UTF-8을 기본 인코딩으로 사용하는 것을 전제로 설계, 설정되는 경우가 많아 레거시와의 충돌이 발생할 수 있다.

이 때문에 Windows 환경에서 프로젝트 경로나 파일 이름에 한글이 포함되어 있으면, MS949를 사용하는 계층과 UTF-8을 기대하는 계층 사이에서 인코딩 불일치가 발생하여 빌드나 실행 오류가 발생할 수 있다. 이러한 문제를 피하기 위해 실무에서는 프로젝트 위치와 이름을 영문/ASCII로 정의하는 것이 일반적이다.

## 빌드하고 실행하기
경로를 찾았다면 명령 프롬프트(cmd)를 열고 스프링 프로젝트 폴더로 이동한 후 `dir` 명령어를 입력해 Gradle 빌드에 필요한 파일이 있는지 확인한다.

이후 `gradlew build` 명령어를 실행한다. BUILD SUCCESSFUL이 출력되면 /build/libs 경로로 이동해 생성된 스냅 샷의 이름을 복사 후 `java -jar XXXspring-0.0.1-SNAPSHOT.jar`와 같이 명령어를 실행하면 cmd에서 스프링 부트 로고가 보이면서 서버가 실행된다.
